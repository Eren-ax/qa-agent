# qa-agent 파트타이머 사용 가이드

채널 파트타이머가 Claude Code를 통해 qa-agent를 실행하여 ALF 품질 검증 리포트를 생성하는 가이드입니다.

## 사전 준비 (처음 한 번만)

### 1. Claude Code 설치

- **Claude Code Desktop App** 설치 (Mac/Windows)
- 또는 **VS Code Extension** 설치
- 또는 **claude.ai/code** 웹 접속

### 2. qa-agent 레포 Clone

Claude Code에서:

```
> ~/qa-agent 경로에 https://github.com/Eren-ax/qa-agent 클론해줘
```

Claude가 자동으로 clone하고 초기 셋업을 진행합니다.

**수동 셋업이 필요한 경우:**

```bash
cd ~/qa-agent
make setup
```

이 명령이 자동으로 수행하는 작업:
- Python 의존성 설치 (`uv sync`)
- Playwright 브라우저 설치 (`playwright install chromium`)

### 3. 환경변수 설정

Claude Code에서:

```
> qa-agent 디렉토리에 .env 파일 만들어줘. ANTHROPIC_API_KEY는 [여기서 받은 키]로 설정해줘
```

**또는 수동으로:**

`~/qa-agent/.env` 파일 생성:

```bash
ANTHROPIC_API_KEY=sk-ant-xxxxx  # Prism Gateway 키
```

키 발급은 AX팀 리드에게 요청하세요.

## 워크플로우: sop-agent → qa-agent

### 전체 흐름

```
[1단계] sop-agent 실행 (별도 가이드 참조)
   ↓
   상담 데이터 분석 결과 (~/sop-agent/results/<고객사>/)
   ↓
[2단계] ALF 세팅 완료 확인
   ↓
   테스트 채널에 지식/규칙/태스크 적용 완료
   ↓
[3단계] qa-agent 실행 ← 이 가이드
   ↓
   품질 검증 리포트 (HTML 슬라이드)
```

### Step 1. sop-agent 완료 확인

qa-agent 실행 전에 다음 파일이 준비되어 있어야 합니다:

```bash
~/sop-agent/results/<고객사>/
├── 03_sop/metadata.json                 # 필수
├── 02_extraction/faq.json               # 필수
├── 02_extraction/patterns.json          # 필수
├── 02_extraction/response_strategies.json  # 권장
└── 04_tasks/TASK*.md                    # 선택
```

**확인 방법 (Claude Code에서):**

```
> ~/sop-agent/results/<고객사> 디렉토리 구조 확인해줘
```

### Step 2. ALF 테스트 채널 준비

1. **테스트 채널 생성**: 프로덕션 채널과 별도로 QA 전용 채널 생성
2. **ALF 세팅 적용**:
   - 지식 문서 업로드 (sop-agent의 `03_sop/*.sop.md` 참고)
   - 규칙 작성 (sop-agent의 `03_sop/*.sop.md` → ALF Rules 섹션)
   - 태스크 세팅 (선택, API 연동이 준비된 경우)
3. **채널 URL 확인**: 예) `https://vqnol.channel.io`

## qa-agent 실행 (Claude Code)

### 기본 실행

Claude Code에서:

```
> <고객사> QA 돌려줘
```

**Claude가 물어보는 질문들:**

1. **채널 URL?**
   ```
   https://<channelId>.channel.io
   ```

2. **sop-agent 결과 경로?**
   ```
   ~/sop-agent/results/<고객사>/
   ```

3. **경쟁사 봇이 작동 중인 고객사인가요?**
   - **Yes** → GL 등 기존 챗봇이 있는 경우 (리포트에 경쟁사 대비 ×N배 표시)
   - **No** → 신규 도입 케이스

4. **ALF 태스크 JSON 있으세요?**
   - 보통은 **No** → `04_tasks/*.md` 자동 파싱
   - 있으면 경로 입력

5. **시나리오 수?**
   - 기본값 **25** 추천 (소요 시간 ~30분)
   - 더 많은 시나리오: 30~50 (소요 시간 ~1시간)

### 실행 단계 (자동)

| Phase | 소요 시간 | 설명 |
|---|---|---|
| 1. Normalize | ~1분 | sop-agent 결과 정규화 |
| 2. Generate scenarios | ~2분 | 시나리오 생성 (Claude가 커버리지 확인 요청) |
| 3. Execute | **30~60분** | Playwright로 ALF와 실제 대화 (가장 오래 걸림) |
| 4. Summarize | 즉시 | 실행 결과 요약 |
| 5. Score | ~5분 | AI Judge 채점 |
| 6. Client report | ~3분 | HTML 리포트 생성 |

**Phase 2 커버리지 확인:**

Claude가 시나리오 생성 후 다음과 같이 물어봅니다:

```
시나리오 25개가 실제 상담의 72.3%를 커버합니다.
계속 진행할까요? (Yes/No)
```

- **72% 이상** → 진행
- **60% 미만** → 시나리오 수 늘리기 (30~35개 권장)

**Phase 3 브라우저 실행:**

- Claude가 백그라운드로 Playwright 실행
- 브라우저 창이 뜨는 것을 보려면: `--headed` 옵션 (디버깅용, 보통은 불필요)

## 결과 확인

### 산출물 위치

```
~/qa-agent/storage/runs/<run_id>/
├── report_client.html    ← **고객사 공유용 슬라이드** (이것만 보면 됨)
├── report.md             ← 내부 상세 리포트 (시나리오별 pass/fail)
└── scores.json           ← 프로그래밍용 데이터
```

`<run_id>` 형식: `<고객사>-<날짜>-<식별자>` (예: `belier-20260417-qa`)

### 고객사 슬라이드 열기

Claude Code에서:

```
> <고객사> QA 리포트 열어줘
```

**또는 수동으로:**

```bash
open ~/qa-agent/storage/runs/<run_id>/report_client.html
```

### 슬라이드 내용

1. **커버 슬라이드**: 고객사명 + Run ID
2. **전체 결과**: Phase 1/Phase 2 관여율 + 테스트 시나리오 수
3. **대화 예시**: ChannelTalk 위젯 UI로 실제 테스트 대화 표시
4. **결론**: 권장 진행 방식 (Phase 1 즉시 도입 → Phase 2 단계적 완료)

**네비게이션:**
- 화살표 키 (← →) 또는 화면 좌우 버튼
- 하단 점으로 슬라이드 위치 확인

## 주요 지표 해석

### Phase 1 vs Phase 2

- **Phase 1 (RAG만)**: 지식 문서 + 규칙만으로 즉시 도입 시 관여율
  - API 연동 없이 **지금 즉시** 가능
  - 보통 60~75% 수준

- **Phase 2 (전체)**: 태스크(API 연동) 포함 시 관여율
  - 주문조회, 취소접수 등 Task 연동 완료 후
  - API 연동 후 **5영업일 내** 완료 가능
  - 보통 80~95% 수준

### 핵심 3대 지표

| 지표 | 의미 | 산출 |
|---|---|---|
| **관여율** | 시나리오가 실제 상담의 몇 %를 대표하는가 | sop-agent 패턴 볼륨 가중 |
| **해결률** | ALF가 대응한 건 중 해결한 비율 | AI Judge 채점 (criterion별 pass/fail) |
| **커버리지** | 실제 상담 중 ALF가 유효 처리한 비율 | 관여율 × 해결률 |

**예시 해석:**

```
Phase 1 관여율: 71.8%
Phase 1 해결률: 88.2%
Phase 1 커버리지: 63.3%
```

→ 실제 상담의 71.8%를 커버하는 시나리오에서 ALF가 88.2% 해결
→ **전체 상담의 63.3%를 ALF가 처리 가능** (지금 즉시)

### 경쟁사 봇 대비

경쟁사 봇(GL 등)이 있는 고객사의 경우:

```
경쟁사 봇 해결률: 12.5%
ALF 해결률: 86.7%
개선: ×6.9배
```

→ **ALF가 기존 봇 대비 6.9배 더 많은 문의를 해결**

## 트러블슈팅

### 1. Phase 3 실행이 멈춤 (시나리오 실행 중)

**증상:**
```
[runner] [5/25] product_003 (persona=vague)
```
이후 30초 이상 진행 없음

**원인:**
- ALF 응답 timeout (60초)
- 채널 로딩 이슈

**해결:**
1. Claude Code에서 중단 (Ctrl+C)
2. 브라우저 창 닫기
3. 재실행:
   ```
   > 같은 run_id로 Phase 3 재실행해줘
   ```

### 2. "ANTHROPIC_API_KEY not set" 오류

**원인:** `.env` 파일 없음 또는 키 오타

**해결:**
```
> ~/qa-agent/.env 파일 확인하고 ANTHROPIC_API_KEY 재설정해줘
```

### 3. Playwright 브라우저 없음

**증상:**
```
playwright._impl._errors.Error: Executable doesn't exist
```

**해결:**
```bash
cd ~/qa-agent
uv run playwright install chromium
```

### 4. 커버리지가 너무 낮음 (<60%)

**원인:** 시나리오 수 부족

**해결:**
```
> 시나리오 35개로 다시 생성해줘
```

더 많은 시나리오 = 더 높은 커버리지 (단, 실행 시간 증가)

### 5. ALF가 전혀 응답하지 않음

**원인:**
- 채널 URL 오류
- 테스트 채널에 ALF 미세팅
- 채널 비공개 설정

**확인:**
1. 브라우저에서 직접 채널 URL 접속
2. ALF 위젯이 뜨는지 확인
3. 직접 메시지 보내서 응답 확인

## 고급 사용법

### 특정 시나리오만 재실행

```
> scenario_id product_001만 다시 실행해줘
```

### 브라우저 보면서 실행 (디버깅)

```
> --headed 모드로 실행해줘
```

브라우저 창이 떠서 ALF와의 대화를 직접 볼 수 있습니다.

### 채점만 재실행 (ALF 세팅 변경 후)

ALF 규칙/지식을 수정한 후 같은 시나리오로 재측정:

```
> 같은 run_id로 Phase 3-5만 재실행해줘
```

### 리포트만 재생성

```
> <run_id> 리포트 재생성해줘
```

## 체크리스트

### QA 시작 전

- [ ] sop-agent 분석 완료 (`~/sop-agent/results/<고객사>/`)
- [ ] 테스트 채널 생성 완료
- [ ] 테스트 채널에 ALF 지식/규칙 세팅 완료
- [ ] 채널 URL 확인 (`https://<channelId>.channel.io`)
- [ ] `~/.qa-agent/.env`에 API 키 설정 완료

### QA 실행 중

- [ ] Phase 2: 시나리오 커버리지 70% 이상 확인
- [ ] Phase 3: 실행 진행 상황 모니터링 (30~60분 소요)
- [ ] Phase 5: 채점 완료, 오류 없음

### QA 완료 후

- [ ] `report_client.html` 브라우저에서 열어서 확인
- [ ] Phase 1/2 관여율 수치 확인
- [ ] 대화 예시 슬라이드 확인 (실제 테스트 대화가 제대로 표시되는지)
- [ ] 고객사 담당자에게 슬라이드 공유
